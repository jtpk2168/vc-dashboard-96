# SaaS Sales Dashboard

A modern, clean, and responsive sales dashboard built with React, Vite, and Tailwind CSS. This dashboard dynamically loads data from a CSV file and provides key performance indicators (KPIs) and a detailed data view.

## ✨ Features

- **Dark Mode Support**: Toggle between sleek High-Contrast and Dark themes with persistent user preference.
- **Interactive Visualizations**: 
  - Revenue Trend (Area Chart)
  - Channel Split (Donut Chart)
  - Product Leaderboard (Bar Chart)
- **Advanced Filtering**: Filter by Product, Acquisition Channel, and Date Range.
- **Dynamic Data Loading**: Fetches and parses sales data from `/public/data/sales_data.csv`.
- **KPI Overview**: Real-time calculation of Revenue, Orders, Profit, and AOV.
- **Premium Aesthetics**: Built with a sleek SaaS design system using Slate and Indigo color palettes.

## 🚀 Technology Stack

- **Framework**: [React](https://reactjs.org/)
- **Build Tool**: [Vite](https://vitejs.dev/)
- **Charts**: [Recharts](https://recharts.org/)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/)
- **Icons**: [Lucide React](https://lucide.dev/)
- **CSV Parsing**: [Papa Parse](https://www.papaparse.com/)

## 🛠️ Getting Started

### Prerequisites

- Node.js (v18 or higher recommended)
- npm or yarn

### Installation

1. Install dependencies:
   ```bash
   npm install
   ```

2. Start the development server:
   ```bash
   npm run dev
   ```

3. Open [http://localhost:5173](http://localhost:5173) in your browser.

### 🤖 AI Setup

To use the **Gemini Strategic Insights** feature:
1. Create a `.env` file in the root directory.
2. Add your Google Gemini API key (get one from [AI Studio](https://aistudio.google.com/)):
   ```env
   VITE_GEMINI_API_KEY=your_key_here
   ```
3. Restart the development server.

### 🗄️ Database Setup (Supabase)

The backend API is configured to read data from a local CSV by default, but is fully ready to connect to **Supabase**.

1. Go to [Supabase](https://supabase.com) and create a new project.
2. In the SQL Editor, create the `sales_data` table by running this SQL snippet:
   ```sql
   CREATE TABLE sales_data (
       id bigint generated by default as identity primary key,
       date date,
       product text,
       channel text,
       orders numeric,
       revenue numeric,
       cost numeric,
       visitors numeric,
       customers numeric
   );
   ```
3. Import your CSV data into this table via the Supabase Table Editor:
   - In the Supabase dashboard, navigate to the **Table Editor** (the grid icon on the left menu).
   - Click on your newly created `sales_data` table.
   - Click the **Insert** button at the top right of the screen, and select **Import data from CSV**.
   - Browse and upload the `public/data/sales_data.csv` file from your local project.
   - Wait for Supabase to auto-map the columns (they should match perfectly).
   - Click **Import Data** at the bottom.
4. Get your API credentials:
   - In the left sidebar of the Supabase dashboard, locate the gear icon (⚙️) at the very bottom and click on **Project Settings**.
   - In the secondary menu under Configuration, click on **API Keys**.
   - Supabase has two key systems. You can use either for `SUPABASE_ANON_KEY`:
     - **New System**: Under the "Publishable and secret API keys" tab, copy the **Publishable key** (starts with `sb_publishable_`).
     - **Legacy System**: Under the "Legacy anon..." tab, copy the **anon / public** key (starts with `ey...`).
   - To find your Project URL, check the **General** tab or the **Data API** link.
5. In your local `.env` file, add your credentials and set the `DATA_SOURCE`:
   ```env
   DATA_SOURCE=supabase
   SUPABASE_URL=your_project_url
   SUPABASE_ANON_KEY=your_anon_key
   ```
6. Restart your development server. The API will now serve data directly from Supabase!
